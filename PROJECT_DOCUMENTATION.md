# توثيق مشروع ArtRights

## أولاً: المشكلة التي يحلها المشروع

المشروع يحل مشكلة حماية حقوق الملكية الفكرية للفنانين (موسيقيين، رسامين، مصورين) من خلال نظام إدارة متكامل. يسمح بتسجيل الأعمال الفنية، تتبع استخدامها غير المصرح به، حساب التعويضات تلقائياً، وإدارة المدفوعات بين الفنانين والوكالات الولائية الحكومية الحكومية.

## ثالثاً: كيف يعمل المشروع

المشروع نظام إدارة حقوق فنية يتكون من 5 رولز رئيسية: Super Admin يدير النظام بالكامل، Admin يدير الوكالة، Gestionnaire يراجع الأعمال الفنية ويدير المدفوعات، Agent ينشئ محاضر الضبط (PV) ويسجل استخدام الأعمال، Artist يسجل أعماله ويدفع ضريبة المنصة. النظام يحسب التعويضات تلقائياً بناءً على معاملات الفئة والجهاز والساعات.

---

## رابعاً: الرولز الأساسية

### 1. Super Admin
يدير النظام بالكامل: إنشاء وإدارة الوكالات، تعيين Admins و Gestionnaires، إدارة الفئات وأنواع الأجهزة، إدارة الشكاوى والتقارير، إدارة القوانين، نقل العمال بين الوكالات.

### 2. Admin
يدير الوكالة: الموافقة/رفض تسجيل الفنانين، إدارة Gestionnaires والـ Agents، إنشاء المهام (Missions)، متابعة الشكاوى والتقارير، عرض المحاضر المالية.

### 3. Gestionnaire
يراجع الأعمال الفنية: الموافقة/رفض الأعمال المرفوعة، إدارة طلبات شحن المحفظة، التحقق من مدفوعات PV وإضافتها لمحفظة الوكالة، إطلاق الأموال للفنانين، إدارة الشكاوى والتقارير.

### 4. Agent
ينشئ محاضر الضبط: إنشاء PV جديد، إضافة الأجهزة والأعمال المستخدمة، حساب الغرامات تلقائياً، تأكيد استلام المدفوعات، إغلاق PV، إرسال الشكاوى والتقارير.

### 5. Artist
يدير أعماله الفنية: رفع الأعمال الفنية، دفع ضريبة المنصة لتفعيلها، شحن المحفظة، متابعة استخدام أعماله في PV، استلام التعويضات، إرسال الشكاوى.

---

## خامساً: تقنية API المستخدمة

**Laravel Sanctum** للمصادقة عبر Bearer Token. الرولز الوحيدان اللذان يتعاملان مع API هما **Agent** و **Artist**. API تدعم: تسجيل الدخول والتسجيل، إدارة الملف الشخصي، إدارة الأعمال الفنية (Artist)، إدارة PVs والمهام (Agent)، إدارة المحفظة، إدارة الشكاوى، إدارة الإشعارات، الوصول للملفات عبر `/api/media/{path}`.

## سابعاً: Diagram Activity لكل رول

### Artist (5 Diagrams)

#### 1. Artist - رفع Artwork جديد
1. Artist يدخل إلى صفحة إضافة Artwork
2. يملأ المعلومات (العنوان، الوصف، الفئة، الملف)
3. النظام يتحقق من صحة البيانات ورفع الملف
4. النظام يحفظ Artwork بحالة PENDING ويحسب ضريبة المنصة تلقائياً
5. النظام يرسل إشعار لـ Gestionnaire للمراجعة

#### 2. Artist - دفع ضريبة المنصة
1. Artist يفتح Artwork بحالة APPROVED
2. يضغط على "دفع ضريبة المنصة"
3. النظام يتحقق من رصيد المحفظة (يجب أن يكون كافياً)
4. يخصم المبلغ من محفظة Artist ويضيفه لمحفظة الوكالة
5. يحدث حالة Artwork إلى PAID ويرسل إشعار للفنان بتفعيل العمل

#### 3. Artist - رفع شكوى
1. Artist يدخل إلى صفحة إنشاء شكوى
2. يختار الوكالة والهدف (Admin أو Gestionnaire)
3. يملأ الموضوع والرسالة ويرفع صور (اختياري)
4. النظام يحفظ الشكوى بحالة PENDING
5. يرسل إشعار للمستهدف وتتغير حالة الشكوى حسب الرد (PENDING → IN_PROGRESS → RESOLVED)

#### 4. Artist - تعديل Artwork
1. Artist يفتح Artwork موجود
2. يضغط على "تعديل" ويغير العنوان أو الوصف أو الملف
3. النظام يحفظ التعديلات
4. يحدث حالة Artwork إلى PENDING (يحتاج موافقة جديدة)
5. يرسل إشعار لـ Gestionnaire للمراجعة من جديد

#### 5. Artist - شحن المحفظة
1. Artist يدخل إلى صفحة المحفظة
2. يضغط على "شحن المحفظة" ويدخل المبلغ وطريقة الدفع
3. يرفع إثبات الدفع (صورة شيك أو تحويل بريدي)
4. النظام يحفظ الطلب بحالة PENDING
5. يرسل إشعار لـ Gestionnaire للموافقة أو الرفض

---

### Agent (5 Diagrams)

#### 1. Agent - فتح PV جديد
1. Agent يدخل إلى إنشاء PV جديد (يمكن ربطه بـ Mission)
2. يملأ معلومات المتجر (الاسم، النوع، التاريخ)
3. يرفع صور الأدلة (حتى 25 صورة)
4. النظام ينشئ PV بحالة OPEN
5. يرسل إشعار لـ Gestionnaire بفتح PV جديد

#### 2. Agent - إضافة Artwork إلى PV
1. Agent يفتح PV بحالة OPEN
2. يختار الوكالة ثم الفنان ثم Artwork (يجب أن يكون APPROVED و PAID)
3. يختار الجهاز (اختياري) وطريقة الحساب (ساعات/عدد) ويدخل القيمة
4. النظام يحسب الغرامة: (معامل الفئة × معامل الجهاز × الساعات × السعر الأساسي)
5. يضيف Artwork للـ PV ويحدث المبلغ الإجمالي ويرسل إشعار للفنان

#### 3. Agent - إغلاق PV
1. Agent يضيف جميع الأجهزة والأعمال للـ PV
2. يتحقق من المبلغ الإجمالي
3. يضغط على "تأكيد استلام الدفع" ويحدد المبلغ المستلم وطريقة الدفع
4. النظام يتحقق من تطابق المبلغ مع الإجمالي بالضبط
5. Agent يضغط على "إغلاق PV" فيصبح CLOSED ويرسل إشعار لـ Gestionnaire

#### 4. Agent - رفع شكوى أو تقرير
1. Agent يدخل إلى صفحة إنشاء شكوى/تقرير
2. يختار النوع (شكوى للـ Admin أو تقرير للـ Gestionnaire)
3. يملأ الموضوع والرسالة ويرفع صور (اختياري)
4. النظام يحفظ الشكوى/التقرير بحالة PENDING
5. يرسل إشعار للمستهدف وتتغير الحالة حسب الرد

#### 5. Agent - تحديث حالة Mission
1. Agent يفتح Mission المخصص له
2. يختار الحالة الجديدة (ASSIGNED → IN_PROGRESS → DONE)
3. إذا كان IN_PROGRESS: يمكن إنشاء PV مرتبط بالـ Mission
4. النظام يحدث حالة Mission
5. يرسل إشعار لـ Gestionnaire بتحديث الحالة

---

### Gestionnaire (5 Diagrams)

#### 1. Gestionnaire - مراجعة وموافقة Artwork
1. Gestionnaire يفتح قائمة الأعمال PENDING
2. يفتح Artwork معين ويراجع الملف والمعلومات
3. يختار الموافقة أو الرفض (مع سبب في حالة الرفض)
4. إذا وافق: يحدث الحالة إلى APPROVED ويحسب ضريبة المنصة
5. يرسل إشعار للفنان بضرورة دفع الضريبة لتفعيل العمل

#### 2. Gestionnaire - الموافقة على طلب شحن المحفظة
1. Gestionnaire يفتح قائمة طلبات شحن المحفظة PENDING
2. يفتح طلب معين ويراجع إثبات الدفع والمبلغ
3. يختار الموافقة أو الرفض مع سبب
4. إذا وافق: يضيف المبلغ لمحفظة الفنان وينشئ Transaction
5. يرسل إشعار للفنان بالموافقة أو الرفض

#### 3. Gestionnaire - التحقق من دفع PV وإضافته لمحفظة الوكالة
1. Gestionnaire يفتح PV مغلق ومدفوع من Agent
2. يتحقق من المبلغ المستلم والمبلغ المطلوب
3. يضغط على "تأكيد الدفع" ويحدد المبلغ
4. النظام يضيف المبلغ لمحفظة الوكالة وينشئ Transaction
5. يحدث حالة PV إلى VALIDATED ويرسل إشعار للـ Agent

#### 4. Gestionnaire - إطلاق الأموال للفنانين
1. Gestionnaire يفتح PV مغلق ومدفوع ومحفظة الوكالة كافية
2. يضغط على "إطلاق الأموال"
3. النظام يحسب المبالغ لكل فنان حسب استخدام أعماله في PV
4. يخصم من محفظة الوكالة ويضيف لمحافظ الفنانين وينشئ Transactions
5. يحدث PV إلى funds_released ويرسل إشعارات للفنانين

#### 5. Gestionnaire - إنشاء Mission
1. Gestionnaire يدخل لإنشاء Mission جديد (يمكن ربطه بشكوى)
2. يختار Agent ويحدد العنوان والموقع والوصف والتاريخ
3. يمكن إضافة رابط خريطة
4. النظام ينشئ Mission بحالة ASSIGNED
5. يرسل إشعار للـ Agent بالمهمة الجديدة

---

### Admin (5 Diagrams)

#### 1. Admin - مراجعة وموافقة تسجيل Artist
1. Admin يفتح قائمة الفنانين PENDING_VALIDATION
2. يفتح حساب فنان معين ويراجع المعلومات ووثيقة الهوية
3. يختار الموافقة أو الرفض مع سبب
4. إذا وافق: ينشئ محفظة للفنان ويحدث الحالة إلى APPROVED ويربطه بالوكالة
5. يرسل إيميل وإشعار للفنان بالموافقة (أو الرفض مع حذف الحساب)

#### 2. Admin - إنشاء Mission
1. Admin يدخل لإنشاء Mission جديد
2. يختار Gestionnaire و/أو Agent ويحدد العنوان والموقع والوصف
3. يحدد التاريخ المقرر (اختياري)
4. النظام ينشئ Mission بحالة ASSIGNED
5. يرسل إشعارات للمستهدفين (Gestionnaire و/أو Agent)

#### 3. Admin - إدارة Agents
1. Admin يفتح قائمة Agents
2. يضغط على "إضافة Agent جديد" أو "حذف Agent"
3. عند الإضافة: يملأ المعلومات (الاسم، الإيميل، رقم الشارة)
4. النظام ينشئ حساب Agent ويربطه بالوكالة
5. يرسل إشعار للـ Admin بنجاح العملية

#### 4. Admin - إدارة Gestionnaires
1. Admin يفتح قائمة Gestionnaires
2. يضغط على "إضافة Gestionnaire جديد" أو "حذف Gestionnaire"
3. عند الإضافة: يملأ المعلومات (الاسم، الإيميل، كلمة المرور)
4. النظام ينشئ حساب Gestionnaire ويربطه بالوكالة
5. يرسل إشعار للـ Admin بنجاح العملية

#### 5. Admin - متابعة الشكاوى والتقارير
1. Admin يفتح قائمة الشكاوى/التقارير الواردة
2. يفتح شكوى/تقرير معين ويراجع المحتوى
3. يمكن الرد مباشرة أو تحويلها لـ Gestionnaire
4. عند الرد: يحدث الحالة إلى RESOLVED
5. يرسل إشعار للمرسل بالرد

---

### Super Admin (5 Diagrams)

#### 1. Super Admin - إدارة الوكالات
1. Super Admin يفتح قائمة الوكالات
2. يضغط على "إنشاء وكالة جديدة" أو "فتح وكالة موجودة"
3. عند الإنشاء: يحدد اسم الوكالة والولاية
4. عند الفتح: يمكن تعيين Admin أو إضافة Gestionnaires/Agents
5. النظام يحفظ التغييرات ويرسل إشعارات للمستخدمين المعنيين

#### 2. Super Admin - تعيين Admin لوكالة
1. Super Admin يفتح وكالة معينة
2. يضغط على "تعيين Admin" أو "إنشاء Admin جديد"
3. يختار Admin موجود أو ينشئ حساب جديد
4. النظام يربط Admin بالوكالة ويحدث agency.admin_id
5. يرسل إيميل للـ Admin بإشعار التعيين

#### 3. Super Admin - إدارة الفئات
1. Super Admin يفتح قائمة الفئات
2. يضغط على "إضافة فئة" أو "تعديل/حذف فئة موجودة"
3. يحدد اسم الفئة ووصفها ومعاملها (coefficient)
4. النظام يحفظ الفئة في قاعدة البيانات
5. يمكن استخدام الفئة الجديدة فوراً في رفع الأعمال

#### 4. Super Admin - نقل العمال بين الوكالات
1. Super Admin يفتح قائمة العمال (Admins, Gestionnaires, Agents, Artists)
2. يختار عامل معين ويفتح صفحة النقل
3. يختار الوكالة الجديدة
4. النظام يحدث agency_id للعامل وجميع سجلاته المرتبطة
5. يرسل إيميل للعامل بإشعار النقل

#### 5. Super Admin - إدارة القوانين
1. Super Admin يفتح صفحة إدارة القوانين
2. يختار اللغة (عربي، إنجليزي، فرنسي)
3. يعدل العنوان والإشعار والأقسام (sections)
4. يمكن إضافة أقسام جديدة مع صيغ حسابية
5. النظام يحفظ التعديلات وتصبح مرئية للجميع
